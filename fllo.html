Lab Challenges & CTF Writeups

Network Reconnaissance using Nmap & dig

Problem Statement: Identifying open ports and services on a target machine, as well as gathering DNS information to understand its network footprint.

Your Approach: I began by using `dig` to perform DNS queries and enumerate subdomains. Following this, I utilized `nmap` for a comprehensive port scan to identify open ports, service versions, and operating system details. I cross-referenced the `nmap` output with `dig` results to build a complete picture of the target.

Tools Used: Wireshark, Burp Suite, dig, nmap

Key Lessons Learned: Solidified my understanding of fundamental network reconnaissance techniques, the importance of combining different tools for thorough analysis, and how to interpret scan results effectively.

Intercepting & Modifying Web Traffic with Burp Suite

Problem Statement: This lab required intercepting HTTP requests and responses from a web application, modifying parameters, and observing the impact on application behavior to identify potential vulnerabilities.

Your Approach: I configured my browser to proxy traffic through Burp Suite. I then navigated to the target web application, intercepting requests to analyze parameters. I experimented with modifying specific parameters, such as user IDs or input values, to bypass client-side validation or access unauthorized resources.

Tools Used: Burp Suite (Community Edition)

Key Lessons Learned: Gained practical experience with web proxy tools, understood the basics of web application security testing, and learned how simple parameter manipulation can lead to vulnerabilities if not properly handled by the server.
